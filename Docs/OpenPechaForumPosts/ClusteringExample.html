The clusters defined by the algorithm have been color-coded in the visualization above. You can see that some of the obvious clusters have been identified relatively well. However, there are some places where the algorithm has defined clusters that seem less than ideal. Below, I've evaluated the silhouette score for the clustering results.

The silhouette score is a metric used to evaluate the quality of clusters in unsupervised clustering. It measures how similar an object is to its own cluster compared to other clusters. The score ranges from -1 to 1, with the following interpretations:

  • 1: The object is well-matched to its own cluster and poorly matched to neighboring clusters (ideal clustering).
  • 0: The object lies on the boundary between clusters.
  • -1: The object is closer to a neighboring cluster than to its own cluster (poor clustering).

The silhouette score for a single data point is calculated as:

$s = \frac{b - a}{\max(a, b)}$

Where:

  • a is the average distance from the point to other points in the same cluster (intra-cluster distance).
  • b is the average distance from the point to points in the nearest neighboring cluster (inter-cluster distance).

The overall silhouette score for the clustering is the mean silhouette score of all points.

Optimize Clustering¶

Below, I've intialized an Optimizer object to aim for a minimum and maximum number of clusters. For this example I've selected a range of 5-15 clusters. Looking at the visualizations above it seems like any greater or fewer than this would be less than ideal. I've also assigned score weights to only include silhouette score and number of clusters in its optimization.

I've then set the optimizer to run 15 trials to try to maximize the silhouette score with the given range of clusters. In general, a larger number of trials will tend to produce better results but this must be balanced against how long we want to run the optimization.

The optimizer will optimize the hyperparameters for UMAP projections ('projection_args') as well as for HDBScan clustering ('clustering args').

Classification Model¶

Preparing Labeled Data¶

Now, that I have clustered the data, I can use the cluster labels as 'true' labels on the data. I can filter outliers from the data (cluster label -1), and then assign the remaining embeddings and labels to X and y variables for supervised classification.

Here I am training the classifier on the embeddings created by MiniLM. This could also be done by creating tf-idf, or any other, encodings. This can be an important decision for classifiers that are intended for use in real time applications because producing embeddings can become computationally expensive.

Note that I have set the 'test_size' for my dataset to 0.8. This is extremely large. More typical values are .2 or .15. However, the classifier consistently performed at nearly 100% accuracy and I wanted to leave room for improvement for the next step in this example.

In [4]:
import xgboost as xgb
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

labeled_pairs = [[i, j] for i, j in zip(cc.embeddings, cc.cluster_labels) if int(j) != -1]

X = [elt[0] for elt in labeled_pairs]
y = [elt[1] for elt in labeled_pairs]

# Split data into training and testing sets
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.8, random_state=0)

Training A Baseline Classifier¶

Below, I train a small XGBoost model to predict the cluster labels.

XGBoost (Extreme Gradient Boosting) is a supervised algorithm based on ensembled decision trees. It uses gradient boosting to optimize the model iteratively during training and implements regularization throughout that process to avoid overfitting.

Despite this, it trains very quickly, allowing us to train multiple version in a short time span to test different hyperparameters.

In [82]:
# Train an XGBoost model
xgb_model = xgb.XGBClassifier(eval_metric='logloss')
xgb_model.fit(X_train, y_train)

# Make predictions
y_pred = xgb_model.predict(X_test)

# Evaluate the model
accuracy = accuracy_score(y_test, y_pred)

This classifier has an accuracy of .85, which is not too bad, but there is clearly room for improvement.

Notably, this classifier also trained in just a minute and a half. This is a small dataset, but that is still much, much faster than training a transformer model for this task. The model itself is also much smaller and therefore much faster in producing predictions.

Like in the clustering steps above, we can improve performance using hyperparameter optimization.

Optimize Classification¶

Below, I've implemented Bayesian optimization using the hyperopt library. Bayesian optimization probabilistically selects new hyperparameters to try based on the outcome of previous training and evaluation loops. This approach is also used by the easy_text_clustering optimizer but the code for this is abstracted in order to be user-friendly.

For this example, I've set the maximum number of trials to 15, but like before this could be set higher if you are willing to wait longer for results.

In [14]:
from hyperopt import fmin, tpe, hp, STATUS_OK, STATUS_FAIL

# Define the search space
space = {
    'max_depth': hp.quniform('max_depth', 2, 8, 1),
    'learning_rate': hp.loguniform('learning_rate', -5, -2),
    'subsample': hp.uniform('subsample', 0.5, 1)
}

def objective(params):
    # Convert max_depth to an integer
    params['max_depth'] = int(params['max_depth'])
    try:
        # Train and evaluate the model
        opt_xgb_model = xgb.XGBClassifier(eval_metric='logloss', **params)
        opt_xgb_model.fit(X_train, y_train)
        y_pred = opt_xgb_model.predict(X_test)
        score = accuracy_score(y_test, y_pred)
        return {'loss': -score, 'status': STATUS_OK}
    except Exception as e:
        # Handle any errors gracefully
        return {'loss': float('inf'), 'status': STATUS_FAIL, 'exception': str(e)}

best_params = fmin(objective, space, algo=tpe.suggest, max_evals=15)

print('Best set of hyperparameters: ', best_params)
